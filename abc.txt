Before you start
If you have important files that aren’t backed up, back them up now (external HDD / cloud) if you can boot into Windows or Safe Mode. Hardware troubleshooting can sometimes trigger data loss.


Have your power adapter connected while troubleshooting.


1) Boot behavior & safe-mode first-aid
Force shutdown by holding the power button until off, then power on and repeatedly press F8 or hold Shift+F8 (some HPs) or hit Esc then F11/F10 during POST to access recovery. If Windows fails to start 2–3 times it should drop you to the Automatic Repair menu.


From Automatic Repair → Advanced options → Troubleshoot → Advanced options → Startup Settings → Restart → choose Safe Mode (option 4) or Safe Mode with Networking (5).


If the BSOD does not occur in Safe Mode, it’s very likely a driver or software problem (not core hardware).


2) Check for recent changes (software/hardware)
Think: did you install new drivers, Windows updates, new RAM, new peripheral, or software just before the BSOD started?


If yes, uninstall that driver/software in Safe Mode: Settings → Apps or Device Manager → right-click device → Uninstall device (check “delete driver software” if prompted).


3) Update / roll back drivers
In Safe Mode (or Normal if stable), open Device Manager (Win+X → Device Manager). Look for devices with a yellow triangle.


Right-click network, display, or storage controllers → Properties → Driver. Try Roll Back Driver if available (if BSOD started after an update). If not, choose Update Driver → Search automatically.


For HP laptops, install drivers using HP Support Assistant or from HP’s support website for your exact model. (If you can’t access web now, prioritize rolling back or uninstalling suspect drivers.)


4) Run system file checks (SFC & DISM)
Open an elevated Command Prompt (search “cmd”, right-click → Run as administrator) and run:
sfc /scannow

Wait until it finishes. If it reports problems it can’t fix, run:
DISM /Online /Cleanup-Image /RestoreHealth

Then re-run sfc /scannow. Reboot after completion.
5) Check disk integrity
In elevated Command Prompt:
chkdsk C: /f /r

It will ask to schedule on next reboot — accept and restart. This checks for bad sectors and file system errors.
6) Memory (RAM) tests — very important for IRQL errors
Run Windows Memory Diagnostic: Win → type Windows Memory Diagnostic → Restart now and check for problems. Let it complete; note any errors.


For a more thorough test, create a MemTest86 USB and run overnight (recommended). If MemTest86 finds errors, you likely have faulty RAM — swap modules / reseat / replace.


7) Analyze crash dumps (optional but helpful)
Crash dumps live in %SystemRoot%\Minidump\ (e.g., C:\Windows\Minidump).


If you want, collect the latest .dmp file and I can help interpret it. (If you prefer a GUI tool on your PC, tools like WinDbg or WhoCrashed can point to the failing driver.)
 — The BSOD referenced ntoskrnl.exe which often means another driver corrupted kernel memory; the dump will show the actual culprit.


8) BIOS / firmware & device firmware
Check BIOS version and update if the laptop BIOS is old. On HP, during boot press Esc then F10 to enter BIOS/UEFI settings. Note the current BIOS version and visit HP Support site for your exact model to compare.


Also update SSD/HDD firmware and major device drivers (chipset, storage, network) from HP if an update is available.


9) Power & overheating / hardware checks
Remove all external devices (USB drives, external GPU, printers) and test. A faulty USB device or driver can cause IRQL errors.


If comfortable, open the laptop and:


Reseat the RAM modules (power off, remove battery if possible, remove and re-insert RAM).


Reseat the SSD/HDD and check cables.


Monitor temps (in BIOS or using HWMonitor in Windows). Overheating can cause instability.


10) Clean boot to isolate software
Run msconfig → Services tab → check Hide all Microsoft services → Disable all → Startup tab → Open Task Manager → disable startup items → Restart.


If stable, re-enable services/startup items in small groups to find the culprit.


11) System Restore or Reset
If the error started recently and you have a restore point: Control Panel → Recovery → Open System Restore → choose a point before the issue.


If nothing else works, you can Reset this PC (Settings → Recovery → Reset this PC) — keep files if you want to preserve data but this may remove apps.


12) When to consider hardware replacement / service
MemTest86 shows errors, or Windows Memory Diagnostic reports errors → RAM likely faulty: replace module(s).


Replacing or reseating RAM eliminates crashes → done.


Persistent crashes after driver / OS reinstall and driver updates → suspect motherboard or CPU. Get HP support or a trusted repair shop to run hardware diagnostics.
